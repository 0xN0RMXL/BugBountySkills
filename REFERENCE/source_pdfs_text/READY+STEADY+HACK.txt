The Recon Process
❖   The main goal of recon is to increase the attack surface
❖   You only need to collect the data that will help you later in the process
❖   If something interesting found, you can go back to recon again, it’s not a one
    time step
❖   Also if you’re stuck with a target you need to go back to the recon step
The Recon Process : DNS Enumeration
❖   Collecting Subdomains
    ➢   Passive
         ■ Amass, Wayback Machine, crt.sh …etc (you can use subenum.sh)
         ■ Chaos (https://chaos.projectdiscovery.io)
         ■ Trickest Inventory (https://github.com/trickest/inventory)
    ➢   Active
         ■ Bruteforcing (dnsx or puredns)
               ● Custom word lists
               ● Find top subdomains (ex: dev.target.com or corp.target.com)
                 ◆ you can use https://github.com/trickest/dsieve
The Recon Process : IP Enumeration
❖   Find the ASNs that belong to an organization using https://bgp.he.net/
❖   Check PTR records to discover more subdomains
❖   Shodan, censys …etc
The Recon Process : HTTP Enumeration
❖   HTTP Information Gathering
    ➢   HTTPX
         ■ For all discovered Subdomains run a single httpx command that will do the following
                ● Grab the status code, response size and page title
                ● Detect technologies
                ● Take a screenshot and save the raw response
    ➢   GoSpider : Spider the domains to find interesting endpoints and JS files.
    ➢   Gau : Passive URL discovery
    ➢   https://urlscan.io/
    ➢   Exposed Api Docs
    ➢   Exposed Docker Images
    ➢   Github and Gitlab
The Recon Process : Dorking and Secrets Scan
❖   You can use chrome “Save All Resources” plugin to fully download a website
    ➢   Scan the saved files using trivy and trufflehog for secrets
    ➢   Analyze the code using codeql
❖   Find github repositories related to the target
    ➢   Scan for secrets using trivy and trufflehog
    ➢   Find open-source projects and analyze them
         ■ Useful to create a custom API list
         ■ Can be scanned using static analysis tools like codeql
The Recon Process : Finding Interesting Domains / Endpoints

❖   You can “grep” for interesting domains, for example
    ➢   Searching by product names : grafana, jenkins, mattermost ..etc.
    ➢   Searching by interesting keywords : admin, dev, sandbox, testnet, blog, docs
❖   You can grep by for interesting files and paths
    ➢   Robots.txt and sitemap.xml
    ➢   /api/ endpoints
    ➢   /graphql endpoints
    ➢   Saml, acs, logout, oauth, redirect
❖   Analyze Javascript to discover endpoints, especially for static websites and
    websites behind a login page
❖   It’s hard to remember all of this, so using a tool like “gf” by tomnomnom can
    be really useful
