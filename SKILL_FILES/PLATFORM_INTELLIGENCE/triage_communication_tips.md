# SKILL: Triage Communication
## Version: 1.0 | Domain: platform-intel

---

## RULES
1. Always professional, even when triager is wrong.
2. Cite evidence (Hacktivity links, CVSS calculator output, tcpdump pcaps).
3. Don't argue severity until bug is accepted.
4. If clarification request → answer all questions in one reply.
5. Don't ping for status before SLA elapsed.

## PHRASES THAT WORK
- "Thanks for the review. I'd like to clarify..."
- "Per the program's stated SLA of [X], could you provide an update?"
- "This appears similar to but distinct from [#hacktivity-link] because..."
- "If helpful I can provide a [video PoC / additional repro / live walkthrough]."

## PHRASES TO AVOID
- "This is obviously..."
- "You should know that..."
- "Your team is incompetent..."
- "I'll publish on Twitter unless..."
- Emojis, exclamation marks, caps lock.

## ESCALATION PATHS
- HackerOne — request mediation via support button on report.
- Bugcrowd — ASR program manager via crowdcontrol-support.
- Intigriti — community manager DM.
- All — public disclosure ONLY after SLA violation + good-faith negotiation.

## DISCLOSURE TIMING
- Wait until program patches and explicitly approves disclosure.
- 90 days post-fix is common standard (Project Zero precedent).
- Some programs require explicit opt-in for disclosure — read program rules.
