# SKILL: Time Management
## Version: 1.0 | Domain: mindset

---

## PRINCIPLE
You will spend more time on recon and re-scoping than on actual exploitation. Plan accordingly.

## DAILY ROUTINE (FULL-TIME HUNTER)
- 06:00–07:00 — read overnight pipeline output, triage new findings
- 07:00–09:00 — deep work on one target's manual hunting
- 09:00–10:00 — break / context switch
- 10:00–12:00 — write reports for verified bugs
- 12:00–13:00 — lunch
- 13:00–16:00 — pivot to second target / fuzz queue / read writeups
- 16:00–17:00 — admin (replies to triage, payouts, learning)

## ROUTINE (SIDE-HUNTER, 10h/WEEK)
- 1h × 5 days: review pipeline output, run targeted tests
- 5h block weekend: deep dive on most promising target

## TIME BOXING
- Recon stage: 4h max per target before forcing pivot to manual hunting.
- Dead-end check: if no signal in 8h on a target, pivot.
- Report writing: 1h per report budget.

## OBSIDIAN WORKFLOW
- One vault, one folder per target.
- Daily note → links to active targets.
- Per-target: scope.md, recon.md, findings.md, drafts.md, submitted.md, payouts.md.
- Tag scheme: `#in-progress`, `#blocked`, `#submitted`, `#paid`, `#duped`.

## CONTEXT-SWITCHING COSTS
- Pin a "current target" on monitor; don't multitask.
- Use Pomodoro for fuzz/wait phases.
- Keep notes verbose — "future you" needs full context to resume.
