# SKILL: Continuous Learning System
## Version: 1.0 | Domain: mindset

---

## THE 5×5 RULE
Each week:
- 5 hours hunting
- 5 PortSwigger labs
- 5 hacktivity reports read deeply
- 5 hours of source code review on a target's open-source dependency
- 5 minutes journaling what's new

## STRUCTURE
```
~/learning/
  ├── 2025-W01/
  │   ├── labs/        — PortSwigger lab solutions, screenshots
  │   ├── writeups/    — H1/BC reports + your annotations
  │   ├── code/        — SCR notes
  │   ├── tools/       — new tools tried + verdict
  │   └── journal.md   — daily 5min notes
  └── ...
```

## SKILL TREE
- Web foundations (req/resp lifecycle, cookies, sessions, CORS, CSP, SOP)
- HTTP exotic (smuggling, request splitting, parser inconsistencies)
- Auth/Authz (OAuth flows, OIDC, SAML, JWT internals)
- Frontend (DOM-based bugs, postMessage, prototype pollution)
- Backend (deserialization, SSTI, SSRF, RCE primitives)
- Cloud (AWS/GCP/Azure IAM, metadata services, k8s, serverless)
- Mobile (Android intents, iOS URL schemes, certificate pinning)
- Web3 (Solidity, ERC standards, common DeFi patterns)
- LLM (prompt injection, RAG, agent attacks)
- Reverse engineering (Ghidra, Frida)
- Programming (deep Python, Go, JS/Node)

## WEEKLY GOAL
- Pick 1 skill from tree.
- Move from "know the name" → "know the patterns" → "know the variants" → "have shipped a finding using it".

## TRACKING
- Spreadsheet: skill | level (1-4) | last touched | next milestone.
- Quarterly self-review: where am I improving / stagnating.
